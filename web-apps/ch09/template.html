§ 9.1 · Key Concept 9.1

Logarithmic Approximation Explorer

The log approximation says the change in ln(x) approximately equals the proportionate change in x. This works well for small changes but breaks down for large ones. Drag the slider to see where the approximation fails.

Key Concept 9.1: The change in ln(x) approximates the proportionate change in x: Δln(x) ≈ Δx/x. Multiplying by 100 gives the percentage change. The approximation is excellent for changes under 10%. For larger changes, use the exact formula: %Δx = 100 × (e^Δln(x) - 1).
Try this
  1. Set the percentage change to 1%. The exact and log approximation are virtually identical — the gap is invisible.
  2. Drag to 10%. The error is about 0.5 percentage points. Still usable but not exact.
  3. Push to 50%. Now the log approximation says 40.5% instead of 50% — a 10 pp error. The approximation breaks down for large changes.

Key Concepts 9.2–9.5: Linear: Δy = β₁Δx (dollar change). Log-linear: %Δy ≈ 100β₁ (semi-elasticity, most common). Log-log: %Δy ≈ β₁%Δx (elasticity, unit-free). Linear-log: Δy ≈ β₁/100 per %Δx. Choose based on theory, data, and interpretation needs.

  3. Compare Log-Log and Linear-Log. Log-Log gives an elasticity of ~1.48. Linear-Log has the worst R².

Key Concept 9.3: In the log-linear model, 100 × β₁ = percentage change in y per unit increase in x. This is the semi-elasticity. The approximation works well for β₁ < 0.10. For larger values, the exact percentage change is 100 × (e^β₁ - 1). The log-linear model is the most common earnings specification because a percentage return scales automatically across income levels.
Try this
  1. Set β₁ = 0.05 (5% return per year). The log-space plot is a gentle line; the level-space curve is nearly straight. The approximation error is tiny.
  2. Set β₁ = 0.13 (the earnings-education estimate). In level space the exponential curve is visible — earnings accelerate for more educated workers.
  3. Push β₁ to 0.25. The level-space curve is steeply exponential. Compare the approximate (25%) vs exact (28.4%) percentage change per unit.
§ 9.2–9.3 · Key Concept 9.4

Elasticity Visualizer

In a log-log model ln(y) = β₀ + β₁ln(x), the slope β₁ is the elasticity: a 1% change in x produces a β₁% change in y. This is unit-free. Adjust the elasticity to see diminishing returns (β₁ < 1) vs. increasing returns (β₁ > 1).

Key Concept 9.4: In the log-log model, β₁ is the elasticity: %Δy / %Δx. If β₁ < 1 there are diminishing returns (concave in levels). If β₁ > 1 there are increasing returns (convex in levels). If β₁ = 1, the relationship is proportional. Elasticity is unit-free — it does not depend on whether you measure in dollars or thousands.
Try this
  1. Set β₁ = 0.33 (typical capital elasticity). In level space the power curve is strongly concave — classic diminishing returns.
  2. Set β₁ = 1.0. The log-space line has slope 1 and the level-space curve becomes a straight line through the origin (proportional).
  3. Set β₁ = 2.0 (increasing returns). The level-space curve becomes convex — quadratic growth. This is rare in economics for production but common for network effects.
§ 9.4 · Key Concepts 9.6, 9.7

Exponential Growth and Rule of 72

Exponential growth in levels becomes linear in logs. The S&P 500 looks like an explosive curve in levels but a steady upward line when logged. The slope directly estimates the average annual growth rate, and the Rule of 72 gives the doubling time.

Key Concepts 9.6–9.7: Exponential growth x_t = x₀(1+r)^t becomes linear in logs: ln(x_t) ≈ ln(x₀) + r×t. The OLS slope on logged data directly estimates the growth rate. The Rule of 72: doubling time ≈ 72/r (where r is in percent). S&P 500 grew at ~6.5%/year (1927-2019), doubling roughly every 11 years.
Try this
  1. Look at Levels + Logs. The left panel shows the explosive S&P 500 curve; the right shows a relatively straight line. This is the power of the log transformation.
  2. Set the custom growth rate slider to 3%. The Rule of 72 says: 72/3 = 24 years to double. At 12%: 72/12 = 6 years to double.
  3. Toggle to Logs Only. Notice the deviations from the trend line — market crashes (1930s, 2000, 2008) show as dips below the linear trend.

Key Concept 9.5: Choosing the right functional form depends on: (1) economic theory — does it predict absolute or percentage effects? (2) data properties — is y right-skewed? (3) model fit — which R² is highest? (4) interpretation needs. The log-linear model typically wins for earnings data because percentage returns scale automatically across income levels.
Try this

  3. Highlight Linear. The raw scatter shows extreme heteroskedasticity — earnings variance grows with education. Logging y stabilizes this.

Key Concepts 9.8–9.9: Log models are essential for cross-country analysis because: (1) variables span huge ranges requiring compression, (2) economic theory predicts multiplicative relationships (production functions), (3) percentage comparisons are more meaningful than absolute ones. The log-log specification directly estimates the output elasticity of capital — a key parameter in growth theory.
