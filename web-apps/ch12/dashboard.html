Key Concept 12.1: When error variance is not constant, default OLS standard errors are invalid. HC1-robust SEs correct this without changing coefficient estimates. Only the SEs, t-statistics, and confidence intervals change. For cross-sectional data, reporting HC1 robust SEs is best practice.
Try this
  1. Toggle between Default and HC1-Robust for the bathrooms variable. The SE ratio is above 1.2 — heteroskedasticity inflates uncertainty for that variable.
  2. Check lotsize — the robust SE is smaller than the default (ratio < 1). This is unusual but possible.
  3. Compare p-values: do any variables switch significance at the 5% level when you change SE type?

Key Concept 12.3: Predicting the average outcome E[y|x*] is more precise than predicting an individual y|x*. The forecast SE satisfies se(y_f) >= s_e — at least as large as the RMSE. As n grows, the CI shrinks to zero, but the PI remains at least +/- 1.96 × s_e wide.
Try this
  1. Set size to the sample mean (~1883 sq ft). The CI is narrowest here. Drag to 3500 — watch both bands widen as you extrapolate.
  2. Toggle to CI Only, then PI Only. The PI dwarfs the CI — most uncertainty is irreducible individual variation.
  3. Note the PI/CI width ratio. It is largest near the mean (where CI is tightest) and decreases at extremes (where estimation uncertainty dominates more).
§ 12.7 · Key Concept 12.8

Type I/II Error Explorer

The power curve shows how the probability of rejecting H₀: β = 0 varies with the true effect size. At β = 0, rejection rate equals α (Type I error). As |β| grows, power rises toward 1. Adjust sample size and SE to see how study design affects detection ability.

Key Concept 12.8: Type I error (false positive) = rejecting a true H₀; probability = α. Type II error (false negative) = failing to reject a false H₀; probability = β. Power = 1 − β. The most powerful test uses the most precise estimator. With n = 30 and σ = 15, you need |β| ≈ 30 to reach 80% power.
Try this
  1. Keep defaults (n=30, σ=15). Read the "80% power at |β|" stat. Now double n to 60 — the minimum detectable effect drops by ~30%.
  2. Increase α from 0.05 to 0.10. Power goes up everywhere, but at the cost of more false positives. This is the fundamental tradeoff.
  3. Set σ = 5 (low noise). The power curve becomes very steep — even small effects are detectable with precise data.

Key Concept 12.2: In time series data, errors are often autocorrelated — today's shock persists into tomorrow. HAC (Newey-West) SEs account for both heteroskedasticity and autocorrelation. Lag length rule of thumb: m = 0.75 × T^(1/3). For T = 241, m ≈ 5.
Try this
  1. Look at lags 1-3 in the ACF — all well above the 95% band. GDP growth has strong persistence.
  2. Increase max lags to 20. By lag 6-7, autocorrelations cross zero and turn slightly negative. Shocks eventually reverse.
  3. In the time series plot, notice how growth clusters: long runs of above-average and below-average periods. This is autocorrelation at work.

Key Concept 12.7: The bootstrap resamples data with replacement many times to estimate the sampling distribution. The percentile CI uses the 2.5th and 97.5th percentiles — no normality assumption needed. With B >= 1000 reps, the bootstrap CI converges to a stable interval.
Try this
  1. Click Run Bootstrap once (200 reps). The histogram is rough. Click again — it smooths out. After 1000+ reps, the CI stabilizes.
  2. Compare the bootstrap SE to the OLS analytical SE. They should be similar — the bootstrap validates the formula.
  3. Notice the histogram may be slightly skewed (not perfectly normal). This is why bootstrap CIs can outperform normal-approximation CIs in small samples.
